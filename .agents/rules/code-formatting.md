**Comments vs. Descriptive names**: Always use descriptive names for functions and variables. Do not write comments, the variable and function names should be descriptive enough. 
**Casing**: Functions should use UpperCamelCase. Variables should use camelCase. Regardless of public, private, global. Godot default functions should be left alone. **Code Blocks**: 
- Functions and statement blocks should not be inline. 
- Functions and statements should have an empty line above and below the block (above the function/statement declaration and below the closing curly brace).
